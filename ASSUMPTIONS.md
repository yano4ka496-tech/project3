# Допущения проекта SafePlant

## Архитектурные допущения

### 1. Offline-first подход
- **Допущение**: Приложение полностью работает без подключения к интернету
- **Обоснование**: Конституция проекта требует offline-first архитектуры
- **Риски**: Ограничения в обновлении данных без установки новой версии APK

### 2. Размер APK < 2 ГБ
- **Допущение**: Размер APK не превышает 2 ГБ после включения всех ресурсов
- **Обоснование**: Требование конституции проекта
- **Риски**: Видео и карты могут увеличить размер APK, что может привести к превышению лимита

### 3. Статические маршруты
- **Допущение**: Все маршруты предустановлены в виде GeoJSON-файлов
- **Обоснование**: Конституция проекта запрещает динамический расчет маршрутов
- **Риски**: Пользователь не может получить альтернативный маршрут, если дорога перекрыта

### 4. QR-привязка без GPS
- **Допущение**: Позиционирование только после сканирования QR-кода
- **Обоснование**: Конституция проекта запрещает использование GPS для позиционирования
- **Риски**: Между сканированиями приложение не отслеживает перемещение пользователя

### 5. Обновление данных через APK
- **Допущение**: Любые изменения данных требуют установки новой версии APK
- **Обоснование**: Конституция проекта запрещает сетевые запросы
- **Риски**: Пользователь может не обновлять приложение долго, что приведет к устареванию данных

## Безопасностные допущения

### 1. Шифрование данных
- **Допущение**: Все чувствительные данные хранятся в зашифрованном виде через Android Keystore
- **Обоснование**: Требование конституции проекта
- **Риски**: Уязвимости в реализации шифрования или ключах

### 2. Проверка root-доступа
- **Допущение**: Приложение обнаруживает root-доступ и предупреждает пользователя
- **Обоснование**: Требование конституции проекта
- **Риски**: Пользователь может игнорировать предупреждение или обойти проверку

### 3. Валидация QR-кодов
- **Допущение**: QR-коды имеют формат "номер_объекта|название" и валидируются перед использованием
- **Обоснование**: Требование конституции проекта
- **Риски**: Некорректные QR-коды могут привести к ошибкам позиционирования

## Бизнес-логические допущения

### 1. Срок действия допуска
- **Допущение**: Допуск действует 30 дней с момента выдачи
- **Обоснование**: Требование конституции проекта
- **Риски**: Пользователь может не пройти квиз вовремя, что заблокирует доступ к опасным зонам

### 2. Повторное прохождение квиза
- **Допущение**: Пользователь может проходить квиз повторно до истечения срока действия текущего допуска
- **Обоснование**: Требование конституции проекта
- **Риски**: Пользователь может злоупотреблять возможностью повторного прохождения

### 3. Блокировка доступа к опасным зонам
- **Допущение**: При истечении срока действия допуска доступ к опасным зонам блокируется, но карта остается доступной
- **Обоснование**: Требование конституции проекта
- **Риски**: Пользователь может продолжить работу с картой без допуска, что может быть небезопасно

## Технические допущения

### 1. Использование MapLibre GL Native
- **Допущение**: Карта отображается с помощью MapLibre GL Native с предустановленными MBTiles
- **Обоснование**: Конституция проекта требует использования офлайн-карт
- **Риски**: Проблемы с производительностью или отображением на некоторых устройствах

### 2. Использование ExoPlayer для видео
- **Допущение**: Видео воспроизводятся через ExoPlayer из локальных файлов
- **Обоснование**: Конституция проекта требует offline-воспроизведения видео
- **Риски**: Проблемы с воспроизведением на некоторых устройствах или версиях Android

### 3. Использование Room с шифрованием
- **Допущение**: База данных Room шифруется через Android Keystore
- **Обоснование**: Требование конституции проекта
- **Риски**: Проблемы с совместимостью или производительностью шифрования

## Ограничения проекта

### 1. Отсутствие аналитики
- **Ограничение**: Приложение не отслеживает использование функций
- **Обоснование**: Конституция проекта требует offline-first подхода
- **Влияние**: Невозможность оптимизации приложения на основе использования

### 2. Отсутствие удаленного отзыва допуска
- **Ограничение**: Потерянный телефон не позволяет отозвать допуск до обновления приложения
- **Обоснование**: Конституция проекта запрещает сетевые запросы
- **Влияние**: Риск безопасности при потере устройства

### 3. Ручное обновление версии
- **Ограничение**: Пользователь должен вручную обновлять приложение
- **Обоснование**: Конституция проекта запрещает сетевые запросы
- **Влияние**: Пользователь может не обновлять приложение вовремя
