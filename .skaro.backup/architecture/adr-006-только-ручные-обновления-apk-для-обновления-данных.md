# ADR-006: Только ручные обновления APK для обновления данных

**Status:** accepted
**Date:** 2026-04-23

## Context
Приложение SafePlant должно обновлять данные (карты, вопросы квиза, маршруты эвакуации). Требуется определить стратегию обновления данных, соответствующую офлайн-первичному подходу.

## Decision
Принято обновлять данные только через полную замену APK. Пользователь должен установить новую версию приложения через Google Play или side-load. Приложение при запуске сравнивает сохранённую версию с текущей (из PackageManager) и при несовпадении сбрасывает цифровой допуск.

## Alternatives
1. **Динамическая загрузка данных через внутреннее хранилище:** rejected because создаёт сложность управления версиями данных, риск повреждения данных при сбоях приложения и не обеспечивает контроля над распространением обновлений.
2. **Использование механизма delta-обновлений:** rejected because увеличивает сложность реализации, создаёт риск неполного обновления данных и не решает проблему необходимости обновления всего приложения при изменении структуры данных.

## Consequences
- Positive: Простота реализации и поддержки, гарантированная целостность данных после обновления, контроль версий данных через версии приложения, автоматический сброс допуска при обновлении (предотвращение использования старых допусков).
- Negative: Необходимость полной переустановки приложения для обновления данных, увеличение размера APK при каждом обновлении, невозможность частичного обновления только изменённых данных, задержка между обновлением данных и их доступностью для пользователей.
- Risks: Пользователи могут долго не обновлять приложение, что приведёт к устареванию данных; невозможность экстренного обновления данных при изменении обстановки на объекте; увеличение нагрузки на сеть при скачивании больших обновлений.
