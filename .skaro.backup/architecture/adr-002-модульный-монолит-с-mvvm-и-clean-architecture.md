# ADR-002: Модульный монолит с MVVM и Clean Architecture

**Status:** accepted
**Date:** 2026-04-23

## Context
Необходимо выбрать архитектурный паттерн для Android-приложения SafePlant, которое должно быть масштабируемым, тестируемым и поддерживаемым в долгосрочной перспективе.

## Decision
Принято использовать модульный монолит с паттерном MVVM (Model-View-ViewModel) и Clean Architecture. Приложение разделено на слои: presentation (UI и ViewModel), domain (бизнес-логика, use cases) и data (репозитории, источники данных). Модули разделены по функциональности (core-navigation, core-database, feature-quiz и т.д.).

## Alternatives
1. **MVI (Model-View-Intent):** rejected because увеличивает сложность реализации для данного типа приложения и не даёт значительных преимуществ по сравнению с MVVM в контексте offline-first приложения.
2. **Классический монолит без чёткого разделения слоёв:** rejected because затрудняет тестирование, переиспользование кода и масштабирование функциональности в будущем.

## Consequences
- Positive: Чёткое разделение ответственности между слоями, улучшаемая тестируемость (особенно бизнес-логики в domain-слое), возможность параллельной разработки разных модулей командой.
- Negative: Увеличение количества boilerplate-кода, необходимость тщательного проектирования интерфейсов между слоями, сложность при внесении изменений в общую архитектуру.
- Risks: Нарушение принципов Clean Architecture может привести к "спагетти-коду" и снижению преимуществ архитектуры; сложность поддержания согласованности между модулями при росте приложения.
