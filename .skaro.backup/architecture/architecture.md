# Architecture

## Overview
**Модульный монолит** (modular monolith) в рамках одного Android-приложения.  
Архитектурный паттерн: **MVVM** с Clean Architecture (слои: presentation, domain, data).  
Полностью офлайн, без серверной части. Обновление данных – только через установку новой версии APK.

## Components

| Компонент | Ответственность |
|-----------|----------------|
| **app** | Точка входа, DI (Dagger Hilt), навигация (Jetpack Navigation Compose) |
| **core‑navigation** | Маршрутизация, deep links из QR‑сканера (открытие карты с позицией) |
| **core‑database** | Room БД, DAO, миграции, шифрование через Android Keystore |
| **core‑storage** | Работа с файлами (MBTiles, GeoJSON, видео, контент), проверка размера (≤2 ГБ) |
| **core‑security** | Шифрование допусков, валидация QR, проверка root (RootBeer, предупреждение) |
| **core‑mapping** | MapLibre GL Native – отрисовка карты, слои (опасные зоны, маршруты, эвакуация) |
| **feature‑quiz** | Квиз: случайная выборка 10 вопросов, подсчёт баллов (8/10), сохранение допуска с датой истечения (30 дней) |
| **feature‑map** | Взаимодействие с картой: отображение предустановленных маршрутов (GeoJSON), опасных зон (полигоны), слоя эвакуации, позиционирование через QR |
| **feature‑training** | Обучающий центр: текст, знаки ТБ, ExoPlayer для 3 видео (из assets, каждое до 50 МБ) |
| **feature‑qr** | Сканер QR (CameraX), парсинг ID зоны, сопоставление с координатами (из Room), вызов навигации на карте |
| **feature‑profile** | Статус допуска (срок 30 дней), сброс при обновлении версии приложения |

## Data Storage

| Тип | Технология | Назначение |
|-----|-----------|------------|
| Векторная карта | MBTiles (конвертирована из PDF через QGIS/ogr2ogr) | Масштабирование от 10×10 км до 10 м, слои: дороги, здания, подписи |
| Пространственные данные (GeoJSON) | Файлы в assets | Опасные зоны (полигоны), предустановленные маршруты (линии по дорогам), точки СИЗ |
| Реляционная БД | Room (SQLite) с шифрованием Keystore | Вопросы квиза, цифровой допуск, соответствие QR‑код → координаты (номер_объекта, название, lat, lon) |
| Файловое хранилище | Внутреннее хранилище Android | Видео (3×50 МБ) – копируются из assets при первом запуске |
| SharedPreferences | EncryptedSharedPreferences | Дата истечения допуска, версия приложения (для сброса при обновлении) |

## Communication
**Отсутствует.** Приложение не выполняет сетевых вызовов (REST, gRPC, брокеры, push).  
Все данные внутри APK или копируются из assets при первом запуске.

## Infrastructure
- **CI/CD**: GitHub Actions – сборка APK/AAB, запуск Detekt, ktlint, unit‑тестов (JUnit5 + Robolectric), интеграционных тестов Room (inMemoryDatabaseBuilder), проверка покрытия (70% для бизнес-логики).
- **Мониторинг**: отсутствует (crash reports не отправляются). Опционально – локальный лог-файл для отладки (не отправляется).
- **Обновления**: ручные – пользователь устанавливает новую версию через Google Play или side-load. Приложение при запуске сравнивает сохранённую версию с текущей (из PackageManager) и при несовпадении сбрасывает цифровой допуск.

## External Integrations
**Нет.** Все библиотеки работают офлайн без внешних сервисов:
- MapLibre GL Native (форк Mapbox GL, без ключей и сетевых тайлов)
- ExoPlayer (без сетевых источников – только локальные файлы)
- CameraX (локальная камера)
- RootBeer (локальная проверка)

## Security

| Аспект | Решение |
|--------|---------|
| Аутентификация | Не требуется (приложение не идентифицирует пользователя) |
| Авторизация | Локальная: проверка наличия и срока действия цифрового допуска (30 дней) при доступе к опасным зонам на карте |
| Защита данных | Шифрование Room через Android Keystore (AES/GCM), EncryptedSharedPreferences для чувствительных данных |
| Валидация ввода | QR-код: проверка формата (например, `123\|Цех А`), санитизация строк, защита от SQL-инъекций (через параметризованные запросы Room) |
| Защита от подделки | Допуск хранится в зашифрованной Room + подпись версией приложения; при обновлении версии допуск сбрасывается |
| Root detection | RootBeer – при обнаружении root показывается предупреждающий диалог, приложение не блокируется |

## Known Trade-offs
1. **Статические маршруты** (предустановленные GeoJSON-линии) – пользователь не может получить альтернативу, если дорога перекрыта; изменение маршрута требует новой версии приложения.
2. **Конвертация PDF → MBTiles** – требуется ручная работа на этапе сборки приложения (QGIS, GDAL), но обеспечивает качественную векторную карту с поддержкой масштабирования до 10 м.
3. **Отсутствие динамической смены данных** – любые изменения на объекте (новые опасные зоны, перекрытия) требуют выпуска новой версии приложения и переустановки.
4. **Размер APK** – видео 150 МБ + карта до 200 МБ + код ≈ 400–500 МБ. Приемлемо для современных устройств (требование ≤2 ГБ соблюдено).
5. **QR-привязка без GPS** – позиционирование только после сканирования; между сканированиями приложение не отслеживает перемещение пользователя.
6. **Нет удалённого отзыва допуска** – компромисс ради offline-first; потерянный телефон не позволяет отозвать допуск до следующего обновления приложения.
7. **Нет аналитики использования** – невозможно отследить, как часто строится маршрут или сдаётся квиз; сделано сознательно из-за полной офлайн-работы.
8. **Ручное обновление версии** – пользователь может не обновлять приложение долго, тогда допуск может истечь, но приложение не напомнит (нет связи). Это документируется в инструкции.
