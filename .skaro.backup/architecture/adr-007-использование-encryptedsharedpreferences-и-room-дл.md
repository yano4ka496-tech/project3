# ADR-007: Использование EncryptedSharedPreferences и Room для безопасности данных

**Status:** accepted
**Date:** 2026-04-23

## Context
Приложение SafePlant должно обеспечивать безопасность данных, особенно цифровых допусков и чувствительной информации о пользователях. Требуется защита данных от несанкционированного доступа, особенно на устройствах с возможностью root-доступа.

## Decision
Принято использовать комбинацию Room с шифрованием через Android Keystore для основной базы данных и EncryptedSharedPreferences для хранения чувствительных метаданных (дата истечения допуска, версия приложения). Дополнительно реализована проверка root-доступа с использованием RootBeer с отображением предупреждающего сообщения при обнаружении.

## Alternatives
1. **Использование обычных SharedPreferences и Room без шифрования:** rejected because не обеспечивает защиту данных при физическом доступе к устройству или root-доступе, что нарушает требования безопасности для приложения, работающего с опасными зонами.
2. **Использование только EncryptedSharedPreferences для всех данных:** rejected because не обеспечивает возможности выполнения сложных запросов, как реляционная база данных, и создаёт сложность в управлении структурированными данными.

## Consequences
- Positive: Высокий уровень защиты данных даже при root-доступе к устройству, возможность выполнения сложных запросов к зашифрованным данным, разделение хранения структурированных и неструктурированных данных, информирование пользователей о потенциальных рисках при root-доступе.
- Negative: Увеличение сложности реализации и отладки, потенциальное снижение производительности запросов из-за шифрования/дешифрования, зависимость от Android Keystore, который может быть недоступен на некоторых устройствах.
- Risks: Возможные проблемы с совместимостью между разными версиями Android при использовании Keystore; сложность миграции данных при смене алгоритма шифрования; риск потери данных при сбоях шифрования/дешифрования; пользователи могут игнорировать предупреждения о root-доступе.
