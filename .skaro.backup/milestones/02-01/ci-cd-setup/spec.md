# Specification: Настройка CI/CD

## Контекст
Необходимость автоматизации сборки, тестирования и проверки качества кода. CI должен соответствовать требованиям конституции (70% покрытия, запуск Detekt/ktlint).

## Функциональные требования
- FR-01: Настроить сборку APK/AAB
- FR-02: Запуск Detekt и ktlint на PR
- FR-03: Запуск unit-тестов (JUnit5 + Robolectric) и интеграционных тестов Room
- FR-04: Генерация отчёта о покрытии (70% для бизнес-логики)
- FR-05: Автоматический запуск скрипта конвертации PDF→MBTiles при изменении PDF
- FR-06: Валидация QR-кодов формата 'номер_объекта|название' в CI
- FR-07: Проверка механизма обнаружения root-доступа в CI
- FR-08: Проверка механизма сброса допуска при обновлении версии в CI
- FR-09: Проверка алгоритма выбора 10 случайных уникальных вопросов в CI
- FR-10: Проверка механизма шифрования Room БД с SQLCipher и Android Keystore в CI

## Критерии приемки
- [ ] Workflow успешно проходит все этапы
- [ ] Отчёты о покрытии доступны в артефактах в форматах HTML и XML
- [ ] Все тесты проходят в CI
- [ ] Скрипт конвертации запускается при изменении PDF
- [ ] В CI проверяется валидация QR-кодов формата 'номер_объекта|название'
- [ ] В CI проверяется механизм обнаружения root-доступа и показа предупреждения
- [ ] В CI проверяется механизм сброса допуска при обновлении версии приложения
- [ ] В CI проверяется алгоритм выбора 10 случайных уникальных вопросов для квиза
- [ ] В CI проверяется механизм шифрования Room БД с использованием SQLCipher и Android Keystore
- [ ] Покрытие бизнес-логики составляет не менее 70%

## Технические детали
- QR-коды должны иметь формат 'номер_объекта|название', где номер_объекта - цифровой идентификатор, название - текст на русском языке
- Проверка root-доступа происходит при запуске приложения, при обнаружении root - показывается предупреждение, но приложение продолжает работу
- Механизм сброса допуска: сравнение versionCode из PackageManager с сохраненным значением, при несовпадении - сброс допуска
- Алгоритм выбора вопросов: выбор с гарантией уникальности вопросов в рамках одной сессии
- Механизм шифрования Room БД: SQLCipher с использованием Android Keystore для управления ключами
- Форматы отчетов о покрытии: HTML для просмотра, XML для интеграции
