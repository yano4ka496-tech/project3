# Specification: Настройка Room с шифрованием

## Context
Необходимость создания зашифрованной базы данных для хранения чувствительных данных (допуски, QR-сопоставления). Шифрование должно использовать Android Keystore.

## Functional Requirements
- FR-01: Создать сущности: QuizQuestion, Permission, QrCodeMapping
- FR-02: Реализовать DAO: QrCodeDao, QuizDao, PermissionDao
- FR-03: Настроить шифрование через SQLCipher с отдельным ключом из Android Keystore (AES/GCM)
- FR-04: Создать EncryptedDatabaseFactory
- FR-05: Реализовать предзагрузку CSV и JSON через createFromAsset()
- FR-06: Обработка некорректных QR-кодов: при сканировании QR-кода с несуществующим номером объекта показывать пользователю сообщение об ошибке и предлагать повторить сканирование
- FR-07: При истечении срока действия цифрового допуска оставлять запись в базе, но помечать как недействительную (isPassValid = 0)
- FR-08: Сбрасывать цифровой допуск при любом обновлении приложения, независимо от срока действия
- FR-09: Реализовать случайный выбор 10 вопросов из квиза без повторений с использованием алгоритма Фишера-Йетса
- FR-10: При обнаружении отсутствующих или поврежденных файлов в assets пропустить их и продолжить работу с доступными файлами
- FR-11: При попытке прохождения квиза без действующего цифрового допуска автоматически перенаправлять пользователя на экран квиза

## Acceptance Criteria
- [ ] База данных зашифрована с использованием SQLCipher и защищена ключом из Android Keystore
- [ ] Данные из CSV/JSON корректно загружены в базу данных
- [ ] DAO протестированы с inMemoryDatabaseBuilder
- [ ] Шифрование работает без потери производительности
- [ ] При сканировании некорректного QR-кода отображается сообщение об ошибке с предложением повторить сканирование
- [ ] Истекшие допуски помечаются как недействительные (isPassValid = 0)
- [ ] При обновлении приложения цифровые допуски сбрасываются (isPassValid = 0) независимо от срока действия
- [ ] Алгоритм Фишера-Йетса для случайной выборки 10 вопросов из квиза работает корректно без повторений
- [ ] При отсутствии или повреждении файлов в assets приложение продолжает работу с доступными файлами
- [ ] При попытке прохождения квиза без действующего допуска пользователь автоматически перенаправляется на экран квиза
- [ ] Логика работы с цифровыми допусками соответствует требованиям истечения срока и обновления приложения
- [ ] Обработка ошибок QR-кодов реализована в соответствии с требованиями
- [ ] Предусмотрена обработка всех возможных ошибок при работе с базой данных

## Verify
- Проверка шифрования базы данных
- Проверка работы с QR-кодами (корректных и некорректных)
- Проверка алгоритма Фишера-Йетса для выбора вопросов
- Проверка сброса разрешений при обновлении приложения
- Проверка автоматического перенаправления на экран квиза при отсутствии допуска
- Unit-тесты базы данных
- Интеграционные тесты Room
- Тесты производительности шифрования
