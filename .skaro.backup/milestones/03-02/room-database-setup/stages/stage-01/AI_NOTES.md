# AI_NOTES — Stage 1: Настройка базы данных с шифрованием

## Что было сделано
- Добавлена зависимость SQLCipher (net.zetetic:android-database-sqlcipher:4.5.4) в build.gradle.kts
- Создан класс EncryptedDatabaseFactory для создания зашифрованной базы данных с использованием ключа из DatabaseKeyManager
- Модифицирован AppDatabase для использования EncryptedDatabaseFactory и добавлены все необходимые сущности и DAO
- Обновлена версия базы данных до 3 с добавлением миграций Migration1To2 и Migration2To3
- Добавлены абстрактные методы для доступа к DAO (qrCodeDao, quizDao, permissionDao)

## Почему такой подход
- Использование SQLCipher с Android Keystore обеспечивает надежное шифрование данных
- EncryptedDatabaseFactory инкапсулирует логику создания зашифрованной базы данных
- Миграции позволяют обновлять структуру базы данных при изменении версии
- Room обеспечивает удобный доступ к данным через DAO и сущности

## Риски и ограничения
- Совместимость SQLCipher с Room может потребовать дополнительных настроек
- Производительность шифрования может повлиять на скорость работы с базой данных
- Ключ шифрования хранится в EncryptedSharedPreferences, что требует надежной защиты устройства

## Соответствие инвариантам
- [ ] Offline-first — база данных полностью локальная и зашифрована
- [ ] Безопасность — используется Android Keystore для шифрования ключа
- [ ] Модульность — шифрование инкапсулировано в отдельный класс
- [ ] Тестируемость — база данных может быть создана с тестовым ключом

## Как проверить
1. Проверить сборку проекта: `./gradlew assembleDebug`
2. Проверить работу шифрования: запустить приложение и проверить, что база данных создается с шифрованием
3. Проверить доступ к DAO: убедиться, что все DAO доступны через AppDatabase
4. Запустить тесты базы данных: `./gradlew test --tests "*DaoTest"`
