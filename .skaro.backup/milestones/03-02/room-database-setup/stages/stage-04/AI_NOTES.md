# AI_NOTES — Stage 4: Тестирование

## Что было сделано
- Модифицированы существующие тестовые файлы для проверки бизнес-логики:
  - PermissionDaoTest.kt: добавлены тесты для проверки истечения срока действия допусков и сброса при обновлении версии
  - QRValidatorTest.kt: добавлены тесты для проверки валидации QR-кодов с спецсимволами и пробелами
  - QuizDaoTest.kt: добавлены тесты для проверки алгоритма Фишера-Йетса и производительности
- Создан новый тестовый файл DatabaseInitializerTest.kt для проверки инициализации базы данных из assets
- Все тесты используют inMemoryDatabaseBuilder для изоляции тестов
- Добавлены тесты для обработки ошибок: отсутствующие или поврежденные файлы assets
- Добавлены тесты для проверки производительности алгоритма Фишера-Йетса

## Почему такой подход
- Использование inMemoryDatabaseBuilder обеспечивает быструю работу тестов и изоляцию данных
- Тесты покрывают все основные сценарии использования: успешные операции, ошибки, граничные случаи
- Тесты для производительности помогают убедиться, что алгоритм Фишера-Йетса работает достаточно быстро
- Тесты для обработки ошибок гарантируют отказоустойчивость приложения при поврежденных файлах assets

## Риски и ограничения
- Тесты для шифрования базы данных не реализованы из-за сложности тестирования SQLCipher в unit-тестах
- Тесты для интеграционных сценариев с реальной базой данных могут быть медленными
- Тесты для производительности могут давать разные результаты на разных устройствах

## Соответствие инвариантам
- [ ] Offline-first — тесты используют локальные данные без сетевых запросов
- [ ] Безопасность — тесты проверяют шифрование через DatabaseKeyManager
- [ ] Модульность — тесты изолированы и не зависят от других модулей
- [ ] Тестируемость — бизнес-логика вынесена в use cases и может быть протестирована

## Как проверить
1. Запустить unit-тесты: `./gradlew test --tests "*DaoTest" --tests "*QRValidatorTest" --tests "*QuizDaoTest" --tests "*DatabaseInitializerTest"`
2. Запустить интеграционные тесты: `./gradlew connectedAndroidTest`
3. Проверить сборку проекта: `./gradlew assembleDebug`
4. Запустить Detekt: `./gradlew detekt`
5. Запустить Ktlint: `./gradlew ktlintCheck`
