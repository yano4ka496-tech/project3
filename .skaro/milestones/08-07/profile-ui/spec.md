# Specification: UI профиля

## Context
Необходимость отображения информации о пользователе и статуса допуска. Профиль должен предоставлять доступ к ключевым функциям.

## Functional Requirements
- FR-01: Показывать статус допуска (действителен до / отсутствует)
  - При истечении срока действия пропуска отображать сообщение 'Пропуск истек' и кнопку 'Пройти квиз заново'
- FR-02: Показывать версию приложения (из PackageManager)
- FR-03: Реализовать кнопку "Пройти квиз заново"
  - Доступна пользователю в любой момент, даже при наличии действующего пропуска
- FR-04: Реализовать кнопку "Ручная позиция"
  - Открывает карту с возможностью выбора точки вручную
- FR-05: Адаптивный дизайн для разных размеров экрана
- FR-06: Экран профиля доступен всегда, даже без действующего пропуска

## Acceptance Criteria
- [ ] Статус допуска отображается верно (действителен до / отсутствует / истек)
- [ ] При истечении срока действия пропуска отображается сообщение 'Пропуск истек'
- [ ] Версия приложения корректна
- [ ] Кнопка "Пройти квиз заново" работает без сбоев и доступна всегда
- [ ] Кнопка "Ручная позиция" открывает карту с возможностью выбора точки вручную
- [ ] Интерфейс адаптивен для разных размеров экрана
- [ ] Экран профиля доступен без действующего пропуска
