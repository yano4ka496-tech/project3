# Specification: Управление версиями

## Context
Необходимость автоматического сброса допуска при обновлении версии приложения. Также требуется возможность ручного сброса. Механизм должен учитывать только версию приложения (versionCode) и не затрагивать данные о прохождении квиза и позиционировании пользователя.

## Functional Requirements
- FR-01: При запуске получать versionCode из PackageManager
- FR-02: Сравнивать с сохраненным в EncryptedSharedPreferences
- FR-03: При различии и наличии действующего допуска очищать таблицу Permission
- FR-04: Реализовать ручной сброс допуска (кнопка в профиле)
- FR-05: При ручном сбросе без изменения версии приложения запрашивать подтверждение у пользователя
- FR-06: Сохранять новую версию после сброса
- FR-07: При недоступности EncryptedSharedPreferences предполагать изменение версии и выполнить сброс
- FR-08: Сброс должен затрагивать только таблицу Permission, не влияя на данные о прохождении квиза и позиционировании

## Acceptance Criteria
- [ ] Автоматический сброс при обновлении работает
- [ ] Ручной сброс работает
- [ ] Версия приложения корректно сохраняется
- [ ] Сброс не влияет на другие данные (квиз, позиционирование)
- [ ] При отсутствии действующего допуска сброс не выполняется
- [ ] При ручном сбросе без изменения версии запрашивается подтверждение
- [ ] При недоступности EncryptedSharedPreferences выполняется сброс допуска
- [ ] После сброса новая версия приложения корректно сохраняется в EncryptedSharedPreferences
