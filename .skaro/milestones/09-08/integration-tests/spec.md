# Specification: Интеграционное тестирование

## Context
Необходимость проверки взаимодействия компонентов приложения. Тесты должны покрывать миграции Room, DAO, use cases и шифрование.

## Functional Requirements
- FR-01: Тесты миграций Room (inMemoryDatabaseBuilder)
- FR-02: Тесты DAO (вставка, обновление, удаление)
- FR-03: Тесты use cases (квиз, допуск)
- FR-04: Тесты шифрования (проверка, что БД зашифрована)
- FR-05: Тесты парсера QR
- FR-06: Тесты обработки сбоя шифрования (переход в режим только для чтения)
- FR-07: Тесты неограниченного количества попыток прохождения квиза
- FR-08: Тесты сброса допусков при обновлении версии приложения
- FR-09: Тесты обработки некорректных QR-кодов (объект не найден)
- FR-10: Тесты истечения срока действия допуска (блокировка доступа)

## Acceptance Criteria
- [ ] Все тесты проходят в CI
- [ ] Покрытие бизнес-логики ≥70%
- [ ] Тесты миграций работают для всех версий
- [ ] Тесты шифрования подтверждают безопасность
- [ ] Тесты обработки сбоя шифрования подтверждают переход в режим только для чтения с предупреждением пользователю
- [ ] Тесты неограниченного количества попыток подтверждают отсутствие ограничений на прохождение квиза
- [ ] Тесты сброса допусков подтверждают, что все допуски автоматически сбрасываются при обновлении версии приложения
- [ ] Тесты обработки некорректных QR-кодов подтверждают отображение ошибки и предложение отсканировать другой QR-код
- [ ] Тесты истечения срока действия допуска подтверждают мгновенную блокировку доступа к опасным зонам с требованием пройти квиз заново
- [ ] Тесты повторного прохождения квиза подтверждают, что пользователь может проходить квиз без ограничений при наличии действующего допуска
