# Specification: Финальная сборка

## Context
Необходимость подготовки релизной версии APK/AAB с проверкой всех требований конституции.

## Functional Requirements
- FR-01: Сборка release APK/AAB
- FR-02: Проверка размера (<2 ГБ)
- FR-03: Проверка отсутствия сетевых запросов (через NetworkSecurityConfig)
- FR-04: Генерация отчета о покрытии (≥70%)
- FR-05: Подготовка AI_NOTES.md и ASSUMPTIONS.md
- FR-06: При истечении срока действия допуска блокировать доступ к опасным зонам, но позволить продолжить работу с картой
- FR-07: При сканировании QR-кода, для которого нет соответствующего объекта в базе данных, отображать ошибку и предлагать отсканировать другой QR-код
- FR-08: Реализовать механизм проверки обновления версии приложения путем сравнения versionCode из PackageManager с сохраненным значением
- FR-09: Запретить выбор маршрута, проходящего через опасную зону без действующего допуска, с уведомлением пользователя
- FR-10: Хранить дату истечения допуска в EncryptedSharedPreferences и проверять ее при каждом доступе к опасным зонам
- FR-11: Разрешить повторное прохождение квиза до истечения срока действия текущего допуска и обновлять срок действия при успешном результате

## Acceptance Criteria
- [ ] APK установлен и работает офлайн
- [ ] Все критерии конституции выполнены
- [ ] Отчет о покрытии доступен
- [ ] Документация обновлена
- [ ] При истечении срока допуска доступ к опасным зонам блокируется, но карта остается доступной
- [ ] При сканировании некорректного QR-кода отображается соответствующее сообщение об ошибке
- [ ] Механизм проверки обновления версии корректно сравнивает versionCode
- [ ] Попытка выбора маршрута через опасную зону без допуска блокируется с уведомлением
- [ ] Дата истечения допуска надежно хранится и проверяется при доступе к опасным зонам
- [ ] Пользователь может повторно пройти квиз и обновить срок действия допуска при успешном результате
