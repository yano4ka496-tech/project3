# Specification: UI-тестирование

## Context
Необходимость проверки ключевых пользовательских сценариев. Тесты должны использовать Espresso и IdlingResource.

## Functional Requirements
- FR-01: Сценарий: сканирование QR → открытие карты → проверка позиции
  - При сканировании QR-кода, не соответствующего формату 'номер_объекта|название', показать диалог с ошибкой и предложить повторить сканирование
  - При сканировании QR-кода в зоне без покрытия GPS, использовать последнюю известную позицию
- FR-02: Сценарий: прохождение квиза → получение допуска
- FR-03: Сценарий: ручное позиционирование на карте
  - При попытке выбора маршрута, проходящего через опасную зону без действующего допуска, автоматически перенаправить на экран прохождения квиза
- FR-04: Сценарий: переключение слоя эвакуации
- FR-05: Использовать IdlingResource для ожидания загрузки карты
- FR-06: Сценарий: истечение срока действия допуска во время использования приложения
  - При истечении срока действия допуска заблокировать доступ к опасным зонам, но разрешить навигацию по безопасным маршрутам
- FR-07: Сценарий: обновление приложения с действующим допуском
  - При обновлении версии приложения сбросить все допуски и перенаправить на экран прохождения квиза
- FR-08: Сценарий: доступ к карте без действующего допуска
  - При попытке доступа к карте без действующего допуска показать диалог с предложением пройти квиз для получения доступа

## Acceptance Criteria
- [ ] UI-тесты проходят на эмуляторе API 29 и 33
- [ ] Все сценарии работают без сбоев
- [ ] IdlingResource корректно обрабатывает загрузку
- [ ] Тесты устойчивы к изменениям UI
- [ ] Для FR-01: корректно обрабатывается некорректный формат QR-кода
- [ ] Для FR-01: корректно обрабатывается отсутствие GPS-покрытия
- [ ] Для FR-03: корректно обрабатывается выбор маршрута через опасную зону без допуска
- [ ] Для FR-06: корректно блокируется доступ к опасным зонам при истечении срока допуска
- [ ] Для FR-07: корректно сбрасываются допуски при обновлении приложения
- [ ] Для FR-08: корректно предлагается пройти квиз при отсутствии доступа к карте

## Open questions
- [ ] Определить точные условия блокировки доступа к опасным зонам при истечении срока допуска
- [ ] Уточнить, как именно будет отображаться диалог с предложением пройти квиз при отсутствии доступа
- [ ] Определить, будет ли сохраняться история прохождения квиза при обновлении приложения
