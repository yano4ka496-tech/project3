# Specification: Логика квиза

## Context
Необходимость проверки знаний пользователя по технике безопасности. Квиз должен случайным образом выбирать 10 вопросов и подсчитывать баллы.

## Functional Requirements
- FR-01: Загружать вопросы из Room (QuizQuestion)
- FR-02: При старте случайным образом выбирать 10 вопросов (детерминированный seed). Если в базе меньше 10 вопросов, вернуть все доступные
- FR-03: Реализовать тип вопросов: одиночный выбор
- FR-04: Подсчитывать правильные ответы
- FR-05: Use case возвращает успех (≥8) или неудачу
- FR-06: Разрешить пересдачу с новыми вопросами каждый раз в течение одного сеанса
- FR-07: Сохранять все результаты прохождения квиза с меткой времени
- FR-08: Варианты ответов хранить в базе данных в формате JSON-массива строк
- FR-09: Считать неотвеченные вопросы как неправильные
- FR-10: Добавить параметр seed в функцию выбора вопросов для детерминированного выбора (для тестирования)

## Data Model
- QuizQuestion:
  - id: Long (первичный ключ)
  - questionText: String (текст вопроса)
  - options: String (JSON-массив строк с вариантами ответов)
  - correctAnswerIndex: Int (индекс правильного ответа в массиве options)
  - category: String (категория вопроса для возможной фильтрации в будущем)

- QuizResult:
  - id: Long (первичный ключ)
  - timestamp: Long (метка времени прохождения)
  - correctAnswers: Int (количество правильных ответов)
  - totalQuestions: Int (общее количество вопросов в тесте)
  - passed: Boolean (результат теста: true если ≥8 правильных ответов)

## Acceptance Criteria
- [ ] Вопросы выбираются случайным образом
- [ ] Подсчёт баллов корректен
- [ ] Use case покрыт unit-тестами
- [ ] При пересдаче вопросы меняются
- [ ] При количестве вопросов меньше 10, выбираются все доступные
- [ ] Неотвеченные вопросы считаются неправильными
- [ ] Результаты сохраняются с меткой времени
- [ ] Для тестирования можно использовать детерминированный seed для воспроизводимости выбора вопросов
- [ ] Варианты ответов хранятся в формате JSON-массива строк
- [ ] При повторном прохождении квиза в одном сеансе генерируется новый набор вопросов
