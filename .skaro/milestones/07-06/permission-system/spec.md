# Спецификация: Система допусков

## Контекст
Необходимость управления цифровыми допусками. Допуск должен действовать 30 дней и сбрасываться при обновлении версии.

## Функциональные требования
- FR-01: При успехе в квизе сохранять запись в зашифрованной Room: timestamp, expiryDate
- FR-02: Реализовать проверку isValid() (expiryDate > now)
- FR-03: При обновлении версии сбрасывать все записи о допуске
- FR-04: Показывать статус допуска в UI
- FR-05: Блокировать доступ к карте при отсутствии допуска
- FR-06: При истечении срока действия допуска во время использования карты показывать диалоговое окно с предложением пройти квиз и блокировать доступ к опасным зонам (показывать опасные зоны, но запретить навигацию через них)
- FR-07: При успешном прохождении квиза заменять старый допуск новым (обновлять дату истечения)
- FR-08: Отображать оставшееся время действия допуска:
  - В формате "Дни и часы" (например, 'Осталось 5 дней 3 часа')
  - При сроке истечения менее часа - в минутах (например, 'Осталось 45 минут')
- FR-09: При обновлении версии приложения, если пользователь находится в процессе использования карты, показывать диалоговое окно с информацией о сбросе и предложением пройти квиз
- FR-10: Проверять валидность допуска при входе в опасную зону на карте (при пересечении границы полигона опасной зоны). Определение позиции осуществляется только через данные от сканера QR-кодов
- FR-11: При сканировании QR-кода, если соответствующий объект не найден в базе данных, показывать сообщение об ошибке и предлагать повторить сканирование
- FR-12: При обновлении версии приложения, если пользователь находится в процессе прохождения квиза, завершить квиз и сбросить прогресс, показав уведомление о сбросе
- FR-13: При выборе маршрута, проходящего через несколько опасных зон, разрешить маршрут, если у пользователя есть действующий допуск (маршрут, который пересекает границы нескольких опасных зон)

## Критерии приемки
- [ ] Допуск сохраняется и истекает через 30 дней
- [ ] Сброс при обновлении работает
- [ ] Статус допуска отображается корректно
- [ ] Интеграция с картой выполнена
- [ ] При истечении срока действия допуска показывается диалоговое окно с предложением пройти квиз и блокируется доступ к опасным зонам (запрет навигации через них)
- [ ] При успешном прохождении квиза старый допуск заменяется новым
- [ ] Отображение оставшегося времени в формате "Дни и часы" корректно, при сроке менее часа - в минутах
- [ ] При обновлении версии приложения во время использования карты показывается диалоговое окно с информацией о сбросе
- [ ] При обновлении версии приложения во время прохождения квиза, квиз завершается и прогресс сбрасывается с уведомлением
- [ ] Проверка валидности допуска выполняется при входе в опасную зону (при пересечении границы полигона)
- [ ] При сканировании несуществующего QR-кода показывается сообщение об ошибке с предложением повторить сканирование
- [ ] Маршруты через несколько опасных зон разрешены при наличии действующего допуска
