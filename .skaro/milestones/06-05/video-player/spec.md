# Specification: Видеоплеер

## Context
Необходимость воспроизведения обучающих видео (до 50 МБ каждое). Видео должны копироваться из assets при первом запуске или при открытии раздела обучения, если видео были удалены вручную.

## Functional Requirements
- FR-01: При первом запуске или при открытии раздела обучения (если видео были удалены вручную) копировать видео из assets во внутреннее хранилище. При обнаружении поврежденного файла, пропустить его и продолжить копирование остальных.
- FR-02: Реализовать ExoPlayer для воспроизведения локальных файлов
- FR-03: Реализовать UI: плейлист, пауза/воспроизведение, прогресс-бар с использованием стандартного ExoPlayer PlayerView
- FR-04: Обрабатывать ошибки: поврежденный файл, не скопирован. Показывать пользователю соответствующие сообщения об ошибках.
- FR-05: Проверять общий размер видео (<150 МБ). Если общий размер превышает 150 МБ, показать пользователю сообщение об ошибке и не копировать видео.
- FR-06: Сохранять информацию о последнем просмотренном видео и позиции воспроизведения в базе данных для возобновления с того же места.

## Acceptance Criteria
- [ ] Видео воспроизводятся без лагов
- [ ] Копирование происходит один раз или при удалении видео вручную
- [ ] UI плеера интуитивно понятен
- [ ] Обработка ошибок работает корректно
- [ ] При обнаружении поврежденного файла, он пропускается, а остальные файлы копируются
- [ ] Если общий размер видео превышает 150 МБ, показывается сообщение об ошибке и копирование не происходит
- [ ] При повторном открытии раздела обучения после удаления видео вручную, видео копируются снова
- [ ] Приложение сохраняет информацию о последнем просмотренном видео и позиции воспроизведения
