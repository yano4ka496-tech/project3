# Specification: Обучающий контент

## Context
Необходимость отображения инструкций по технике безопасности. Контент должен быть организован по разделам с возможностью поиска. Контент хранится локально в assets приложения.

## Functional Requirements
- FR-01: Загружать инструкции из assets (JSON для метаданных и Markdown для контента)
- FR-02: Реализовать гибкую иерархию разделов с возможностью вложенных подразделов (общие правила, ЧС, штрафы, знаки и др.)
- FR-03: Реализовать поиск по ключевым словам без учета регистра
- FR-04: Реализовать закладки (локальное сохранение в базе данных) с возможностью поиска и фильтрации
- FR-05: Реализовать responsive design с гибкими компонентами, адаптирующимися под размер экрана
- FR-06: Обрабатывать отсутствие контента в assets через отображение заглушки с сообщением

## Acceptance Criteria
- [ ] Контент корректно отображается из JSON и Markdown файлов
- [ ] Поиск работает без учета регистра
- [ ] Закладки сохраняются между сессиями в базе данных
- [ ] Интерфейс адаптивен через responsive design
- [ ] При отсутствии контента отображается соответствующая заглушка
- [ ] Гибкая иерархия разделов корректно отображает вложенные подразделы
- [ ] Закладки можно искать и фильтровать
