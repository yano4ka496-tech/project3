# Specification: Система безопасности

## Context
Необходимость защиты от некорректных входных данных и обнаружения root-доступа. QR-коды должны строго соответствовать формату.

## Functional Requirements
- FR-01: Реализовать парсер QR: формат "номер_объекта|название"
- FR-02: Добавить валидацию формата QR (регулярное выражение)
- FR-03: Реализовать санитизацию входных строк
- FR-04: Интегрировать RootBeer для обнаружения root
- FR-05: Показывать предупреждающий диалог при обнаружении root
- FR-06: Использовать EncryptedSharedPreferences для даты допуска
- FR-07: При истечении срока действия пропуска немедленно ограничить доступ к опасным зонам

## Acceptance Criteria
- [ ] Некорректные QR отклоняются с сообщением об ошибке
- [ ] После показа ошибки некорректного QR-кода приложение автоматически возвращается к экрану сканера
- [ ] Root-предупреждение показывается при обнаружении root
- [ ] При обнаружении root функциональность приложения ограничена (только доступ к опасным зонам)
- [ ] EncryptedSharedPreferences корректно хранит данные (дата истечения пропуска и версия приложения)
- [ ] Система устойчива к SQL-инъекциям
- [ ] 'номер_объекта' содержит только цифры и буквы, максимальная длина 20 символов
- [ ] 'название' содержит только цифры и буквы, максимальная длина 50 символов
- [ ] При ошибке парсинга QR-кода показывается сообщение об ошибке
- [ ] При ошибке парсинга QR-кода предоставляется возможность повторного сканирования
- [ ] При истечении срока действия пропуска доступ к опасным зонам немедленно ограничивается
- [ ] Если пропуск истекает, пока пользователь уже находится в опасной зоне, система заставляет его немедленно покинуть зону

## Validation Rules
- 'номер_объекта' должен соответствовать шаблону: `[A-Za-z0-9]{1,20}`
- 'название' должно соответствовать шаблону: `[A-Za-z0-9]{1,50}`
- QR-код должен содержать ровно один символ `|` в качестве разделителя

## Sanitization Rules
- Входные строки из QR-кода должны быть обработаны с экранированием специальных символов с использованием стандартного экранирования

## Data Storage
- EncryptedSharedPreferences должен хранить:
  - Дата истечения пропуска (в виде timestamp) по ключу 'pass_expiration'
  - Версия приложения (для проверки совместимости) по ключу 'app_version'

## Root Detection Behavior
- При обнаружении root-доступа:
  - Показать предупреждающий диалог с информацией о рисках
  - Ограничить доступ к опасным зонам (критические функции)
  - Продолжить работу приложения с ограниченным функционалом (доступ к безопасным зонам разрешен)

## Access Control
- При истечении срока действия пропуска:
  - Немедленно проверить дату истечения при каждом входе в опасную зону
  - Отклонить доступ к опасным зонам с соответствующим сообщением
  - Продолжить работу в безопасных зонах приложения
  - Если пользователь уже находится в опасной зоне при истечении срока, система заставляет его немедленно покинуть зону
