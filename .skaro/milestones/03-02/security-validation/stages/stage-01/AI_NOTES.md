# AI_NOTES — Stage 1: Улучшение валидации QR-кодов

## Что было сделано
- Реализована строгая валидация QR-кодов с использованием регулярных выражений
- Добавлена санитизация входных строк с экранированием специальных символов
- Обновлен интерфейс сканера для отображения ошибок через Toast
- Реализован автоматический возврат к сканированию после ошибки без дополнительных действий пользователя
- Улучшена обработка как сканированных, так и вручную введенных QR-кодов

## Почему такой подход
- Использование регулярных выражений обеспечивает строгую проверку формата QR-кода
- Санитизация с экранированием специальных символов защищает от инъекций и некорректных данных
- Toast-сообщения не блокируют интерфейс и автоматически исчезают
- Автоматический возврат к сканированию улучшает пользовательский опыт
- Сохранена имитация сканирования для тестирования

## Риски и ограничения
- Санитизация может изменять формат QR-кода, но это необходимо для безопасности
- Имитация сканирования заменена на реальную логику в продакшене
- Toast-сообщения могут быть пропущены пользователем при быстром сканировании
- Требуется тестирование с различными типами некорректных QR-кодов

## Как проверить
1. Запустить приложение и проверить валидацию QR-кодов:
   - Ввести корректный QR-код (например, "123|ЦехА") — должен перейти на карту
   - Ввести некорректный QR-код (например, "123|Цех@") — должен показать ошибку и продолжить сканирование
   - Ввести QR-код с некорректным форматом (например, "123|") — должен показать ошибку
2. Проверить автоматический возврат к сканированию после ошибки:
   - После показа ошибки сканер должен продолжать работать без перезапуска
3. Проверить ручной ввод:
   - Ввести некорректный QR-код в поле ручного ввода — должен показать ошибку и позволить ввести снова
