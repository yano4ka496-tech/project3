# AI_NOTES — Stage 2: Настройка базы данных и шифрования

## Что было сделано
- Реализована основная база данных AppDatabase с поддержкой шифрования через SQLCipher
- Создан DatabaseKeyManager для управления ключами шифрования с использованием Android Keystore
- Настроены конвертеры для Room для преобразования типов данных
- Реализован SQLCipherOpenHelperFactory для создания зашифрованной базы данных
- Добавлена поддержка WAL режима для улучшения производительности

## Почему такой подход
- Использование SQLCipher с Android Keystore обеспечивает высокий уровень безопасности данных
- WAL режим улучшает производительность операций записи в базу данных
- DatabaseKeyManager автоматически генерирует и хранит ключи шифрования
- Конвертеры позволяют использовать типы Date в сущностях Room

## Риски и ограничения
- Возможные проблемы с совместимостью SQLCipher и Room на некоторых устройствах
- При сбросе устройства ключ шифрования будет утерян, что приведет к потере данных
- Размер базы данных может увеличиться из-за шифрования
