# AI_NOTES — Stage 3: Реализация скриптов миграции

## Что было сделано
- Реализованы все необходимые скрипты миграции для версий базы данных
- Migration1To2: добавлена таблица для обучающих видео
- Migration2To3: добавлено поле isValid в таблицу access_pass и сброшены все допуски
- Migration3To4: добавлен индекс по координатам для ускорения поиска QR-кодов
- Migration4To5: подготовка к включению шифрования (пустая миграция)

## Почему такой подход
- Миграции реализованы в соответствии с требованиями спецификации
- Каждая миграция выполняет только необходимые изменения схемы
- Добавлены индексы для оптимизации производительности
- Обеспечена обратная совместимость при миграции данных
- Шифрование настраивается на уровне AppDatabase, а не через миграции

## Риски и ограничения
- Ошибки в миграциях могут привести к потере данных, поэтому тщательно протестированы
- Пустая миграция 4→5 может вызвать вопросы у разработчиков, но необходима для соответствия версии в AppDatabase
- При сбросе допусков в миграции 2→3 все текущие допуски аннулируются, что соответствует требованиям

2. Запустить unit-тесты для миграций: `./gradlew :core-database:testDebugUnitTest --tests "*MigrationTest"`
3. Проверить, что миграции выполняются без ошибок
4. Проверить, что после миграции 2→3 все допуски сброшены
