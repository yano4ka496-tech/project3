# AI_NOTES — Stage 6: Автоматизация проверок

## Что было сделано
- Создан скрипт `scripts/run_all_checks.sh` для автоматизации всех проверок модуля `core-database`
- Скрипт последовательно выполняет все проверки из раздела "Verify" и прерывается при первой ошибке
- Реализована проверка покрытия тестами через Jacoco
- Добавлена проверка линтера Detekt
- Включена проверка сборки проекта

## Почему такой подход
- Раздел "Verify" содержит 6 команд, которые необходимо выполнить для проверки качества модуля
- Автоматизация проверок через скрипт позволяет быстро и последовательно выполнять все проверки
- Скрипт возвращает код ошибки, если какая-либо проверка не прошла, что удобно для CI/CD
- Последовательное выполнение проверок позволяет выявить все проблемы в коде

| Файл | Действие | Описание |
|---|---|---|
| `scripts/run_all_checks.sh` | Создан | Скрипт для автоматизации проверок модуля `core-database` |

## Риски и ограничения
- Скрипт предполагает, что он запускается из корневой директории проекта
- Скрипт не проверяет покрытие тестами (только генерирует отчет)
- Скрипт не открывает отчет о покрытии тестами, только генерирует его
- Время выполнения всех проверок может быть значительным (особенно интеграционные тесты)

## Соответствие инвариантам
- [ ] Структура проекта — соблюдена (скрипт добавлен в директорию `scripts/`)

## Как проверить
1. Запустить скрипт: `./scripts/run_all_checks.sh`
2. Проверить, что все проверки прошли успешно
3. Проверить, что отчет о покрытии тестами сгенерирован в `core-database/build/reports/jacoco/test/html/index.html`
4. Убедиться, что отчет линтера Detekt сгенерирован в `core-database/build/reports/detekt/`
