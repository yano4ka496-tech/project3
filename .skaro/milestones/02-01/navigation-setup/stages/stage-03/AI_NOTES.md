# AI_NOTES — Stage 3: Обработка ошибок и управление состоянием навигации

## Что было сделано
- Реализована детальная обработка ошибок навигации в ErrorHandler
- Добавлена логика проверки существования зон и действующих допусков
- Обновлен NavigationActions для работы с проверкой доступа
- Реализован QRScannerViewModel с обработкой сканированных QR-кодов
- Добавлен ProfileViewModel с проверкой версии приложения и сбросом допусков
- Обновлен QRValidator с дополнительной валидацией ID зон

## Почему такой подход
- Использован централизованный обработчик ошибок ErrorHandler для единообразия обработки ошибок
- Логика проверки доступа вынесена в отдельные функции для переиспользования
- Для навигации используется NavigationActions с интеграцией Hilt
- Проверка версии приложения при запуске для сброса допусков при обновлении
- Валидация QR-кодов на нескольких уровнях: формат, парсинг, существование в базе

## Риски и ограничения
- Сложность управления состоянием при навигации, особенно при обработке deep links
- Возможные конфликты при одновременном обновлении версии и работе с допусками
- Тестирование всех сценариев навигации может быть сложным
- Использование runBlocking в основном потоке может блокировать UI при операциях с базой данных

## Соответствие инвариантам
- [ ] Модульная архитектура — соблюдается, каждый модуль имеет четкие обязанности
- [ ] Offline-first — соблюдается, все операции выполняются локально
- [ ] Безопасность данных — соблюдается, используется шифрование через Keystore
- [ ] Максимальная длина функции — соблюдается, функции не превышают 40 строк
- [ ] Максимальная глубина вложенности — соблюдается, не превышает 4 уровня
- [ ] Тестирование — добавлена логика для тестирования бизнес-логики

## Как проверить
1. Проверка сборки приложения: `./gradlew assembleDebug`
2. Проверка навигации между экранами: `./gradlew connectedAndroidTest --tests "com.safeplant.MainActivityTest"`
3. Проверка deep links: `adb shell am start -W -a android.intent.action.VIEW -d "safeplant://map?zoneId=123" com.safeplant`
4. Проверка линтинга: `./gradlew ktlintCheck`
5. Проверка детекта: `./gradlew detekt`
