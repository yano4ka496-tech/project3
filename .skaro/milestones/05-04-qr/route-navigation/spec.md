# Specification: Навигация по маршрутам

## Context
Необходимость отображения предустановленных маршрутов на карте. Маршруты должны проверяться на наличие допуска.

## Functional Requirements
- FR-01: Реализовать выбор цели из списка
- FR-02: Подсвечивать соответствующий маршрут из routes.geojson
- FR-03: Отображать направление движения стрелками вдоль линии маршрута с указанием направления
- FR-04: Блокировать маршрут при отсутствии допуска через опасную зону и перенаправлять на экран прохождения квиза
- FR-05: Реализовать сброс маршрута по кнопке с полной очисткой карты и возвратом к начальному состоянию
- FR-06: При попытке выбора маршрута, не предустановленного в GeoJSON, показывать ошибку 'Маршрут не найден'
- FR-07: При обновлении версии приложения во время активного маршрута автоматически сбрасывать маршрут и требовать повторный сканирование QR

## Acceptance Criteria
- [ ] Маршрут подсвечивается на карте
- [ ] Блокировка при отсутствии допуска работает и перенаправляет на квиз
- [ ] Направление движения отображается стрелками вдоль линии маршрута
- [ ] Сброс маршрута работает без сбоев и возвращает к начальному состоянию
- [ ] При выборе несуществующего маршрута отображается ошибка 'Маршрут не найден'
- [ ] При обновлении приложения во время активного маршрута маршрут сбрасывается
- [ ] Алгоритм проверки пересечения маршрута с опасными зонами проверяет все точки маршрута на расстояние до ближайшей опасной зоны

## Implementation Details
- Для отображения направления движения использовать стрелки вдоль линии маршрута
- При отсутствии допуска через опасную зону автоматически перенаправлять на экран прохождения квиза
- При обновлении версии приложения во время активного маршрута автоматически сбрасывать маршрут и требовать повторный сканирование QR
- Для проверки пересечения маршрута с опасными зонами использовать алгоритм проверки всех точек маршрута на расстояние до ближайшей опасной зоны
- При выборе маршрута, не предустановленного в GeoJSON, показывать ошибку 'Маршрут не найден'
- Сброс маршрута по кнопке должен полностью очищать карту и возвращать к начальному состоянию
