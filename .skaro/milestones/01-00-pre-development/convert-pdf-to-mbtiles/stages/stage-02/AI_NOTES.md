# AI_NOTES — Stage 2: Реализация навигации и интеграция QR-сканера

## Что было сделано
- Реализован экран QRScannerScreen с использованием CameraX для сканирования QR-кодов
- Реализован экран MapScreen с интеграцией MapLibre GL для отображения карты
- Реализованы экраны QuizScreen, TrainingScreen и ProfileScreen с базовой навигацией
- Реализован класс MapRenderer для работы с картой и позиционированием
- Модифицирован AppNavigation для интеграции проверки root-доступа и навигации между экранами
- Добавлены необходимые зависимости для CameraX и MapLibre GL

## Почему такой подход
- Использование CameraX обеспечивает современный и надежный способ работы с камерой
- MapLibre GL выбран для офлайн-отображения карт с поддержкой векторных тайлов
- Навигация реализована через Jetpack Navigation Compose, что соответствует современным практикам
- Проверка root-доступа выполняется при запуске приложения для безопасности
- Обработка ошибок QR-сканирования реализована через диалог с предложением повторить

## Риски и ограничения
- Интеграция CameraX с Compose может потребовать дополнительных настроек
- MapLibre GL может иметь ограничения на производительность на старых устройствах
- Реальное сканирование QR-кодов требует использования библиотеки (например, ML Kit или ZXing), в текущей реализации используется имитация
- Проверка root-доступа не является 100% надежной на всех устройствах
