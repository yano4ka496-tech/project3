# Specification: Подготовка вопросов квиза

## Context
Необходимость создания базы вопросов для проверки знаний по технике безопасности. Вопросы должны покрывать ключевые аспекты безопасности на объекте.

## Functional Requirements
- FR-01: Создать JSON-файл с вопросами в assets
- FR-02: Структура вопроса: {id, question, options, correctAnswer}
- FR-03: Минимум 50 вопросов для случайной выборки
- FR-04: Вопросы охватывают: общие правила, ЧС, штрафы, знаки
- FR-05: Разместить в `app/src/main/assets/quiz/`
- FR-06: Сохранять все результаты прохождения квиза с уникальными ID
- FR-07: Всегда проводить все 10 вопросов квиза без преждевременного завершения
- FR-08: Обрабатывать отсутствие файла questions.json в assets путем показа сообщения об ошибке и закрытия квиза
- FR-09: Предоставлять доступ к квизу без проверок действующего пропуска
- FR-10: При наличии в файле менее 50 вопросов, выбирать все доступные вопросы (меньше 10)

## Acceptance Criteria
- [ ] Все вопросы имеют уникальные ID
- [ ] Формат JSON соответствует структуре
- [ ] Вопросы покрывают все требуемые темы
- [ ] Все результаты прохождения квиза сохраняются с уникальными ID
- [ ] Квиз всегда проходит все 10 вопросов без преждевременного завершения
- [ ] При отсутствии файла questions.json приложение показывает сообщение об ошибке и закрывает квиз
- [ ] К доступен без проверок действующего пропуска
- [ ] При наличии менее 50 вопросов в файле, выбираются все доступные вопросы
