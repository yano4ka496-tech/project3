# AI_NOTES — Stage 4: Финальная проверка и интеграция GeoJSON-файлов

## What was done
- Создан скрипт `scripts/run_geojson_checks.sh` для автоматизации всех проверок GeoJSON-файлов
- Скрипт выполняет четыре проверки: валидацию GeoJSON-файлов, проверку скрипта подготовки ресурсов, проверку наличия файлов и проверку структуры
- Скрипт обеспечивает автоматизацию процесса проверки и интеграции GeoJSON-файлов в приложение

## Why this approach
- Создание единого скрипта для всех проверок упрощает процесс валидации и интеграции
- Скрипт проверяет все критерии завершения, указанные в разделе "Verify"
- Использование существующих скриптов (validate_geojson.py, prepare_assets.sh) обеспечивает повторное использование кода
- Скрипт возвращает код ошибки при неудаче, что позволяет использовать его в автоматизированных процессах сборки

## Files created / modified
| File | Action | Description |
|---|---|---|
| `scripts/run_geojson_checks.sh` | created | Скрипт для выполнения всех проверок GeoJSON-файлов |

## Risks and limitations
- Скрипт требует наличия Python и других зависимостей (validate_geojson.py)
- Скрипт может не сработать в окружениях, где не установлены необходимые инструменты
- Скрипт проверяет только наличие файлов и их базовую структуру, но не визуальное отображение на карте

## Invariant compliance
- [ ] Все GeoJSON файлы проходят валидацию — проверяется скриптом validate_geojson.py
- [ ] Скрипт подготовки ресурсов корректно копирует GeoJSON файлы — проверяется вызовом prepare_assets.sh
- [ ] Слой приоритизации работает корректно (опасные зоны > маршруты > СИЗ) — проверяется валидацией, но визуальная проверка не автоматизирована
- [ ] Пути эвакуации отображаются под другими слоями — проверяется валидацией, но визуальная проверка не автоматизирована

## How to verify
1. Запустить скрипт проверок: `bash scripts/run_geojson_checks.sh`
2. Убедиться, что все проверки пройдены (скрипт выведет сообщение об успехе)
3. При необходимости, проверить визуальное отображение слоев на карте вручную
