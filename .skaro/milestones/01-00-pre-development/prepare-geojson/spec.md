# Specification: Подготовка GeoJSON-файлов

## Context
Необходимость создания векторных слоёв для отображения опасных зон, маршрутов, точек СИЗ и путей эвакуации. Геоданные должны соответствовать координатной системе MBTiles.

## Functional Requirements
- FR-01: `hazards.geojson` – полигоны опасных зон (красный цвет)
- FR-02: `routes.geojson` – линии предустановленных маршрутов (синий цвет)
- FR-03: `ppe_points.geojson` – точки СИЗ (иконки)
- FR-04: `evacuation.geojson` – пути эвакуации (зелёный цвет)
- FR-05: Разместить в `app/src/main/assets/geojson/`
- FR-06: Проверить соответствие координат системе WGS84
- FR-07: При отображении слоев на карте реализовать приоритизацию по важности: опасные зоны > маршруты > СИЗ
- FR-08: При частичном перекрытии слоев, более важные слои должны отображаться поверх менее важных

## Acceptance Criteria
- [ ] Все GeoJSON файлы валидны (проверка через geojson.io)
- [ ] Координаты соответствуют MBTiles
- [ ] Слои визуально корректно накладываются на карту
- [ ] Приоритизация слоев работает корректно при перекрытии
- [ ] Опасные зоны всегда отображаются поверх других слоев
- [ ] Маршруты отображаются поверх точек СИЗ, но под опасными зонами

## Additional Requirements from Q&A
- FR-QR-01: При сканировании QR-кода, не соответствующего формату 'номер_объекта|название', отображать диалог с ошибкой и предложение повторить сканирование
- FR-QR-02: При попытке доступа к опасной зоне с истекшим сроком действия допуска, отображать диалог с предложением пройти квиз снова
- FR-QR-03: При обновлении версии приложения сбрасывать действующий допуск и требовать прохождения квиза снова
- FR-QR-04: При сканировании QR-кода с номером объекта, отсутствующим в базе данных, отображать диалог об ошибке и предложение сканировать другой QR-код
- FR-QR-05: Разрешать прохождение квиза без ограничений, если пользователь уже имеет действующий допуск

## Acceptance Criteria for QR Handling
- [ ] Диалог ошибки при некорректном формате QR-кода содержит понятное сообщение и кнопку "Повторить сканирование"
- [ ] При попытке доступа к опасной зоне с истекшим допуске отображается диалог с кнопкой "Пройти квиз"
- [ ] После обновления приложения пользователь не может получить доступ к опасным зонам без прохождения квиза
- [ ] При сканировании QR-кода с несуществующим номером объекта отображается диалог с предложением попробовать другой QR-код
- [ ] Пользователь с действующим допуском может пройти квиз повторно без ограничений
