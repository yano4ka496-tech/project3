# AI_NOTES — Stage 2: Реализация конкретных слоев карты

## What was done
- Реализован слой опасных зон (HazardsLayer) с красными полигонами
- Реализован слой предустановленных маршрутов (RoutesLayer) с синими линиями
- Реализован слой эвакуации (EvacuationLayer) с зелеными линиями и проверкой доступа
- Реализован слой точек СИЗ (PPEPointsLayer) с желтыми иконками
- Добавлен класс PopupInfo для информации всплывающих окон
- Реализована логика загрузки данных из GeoJSON-файлов
- Добавлена обработка нажатий на элементы слоев
- Реализована видимость слоев в зависимости от масштаба

## Why this approach
- Каждый слой наследуется от BaseLayer, что обеспечивает единый интерфейс и общую функциональность
- Используется единый подход к парсингу GeoJSON для всех слоев
- Слой эвакуации имеет дополнительную проверку доступа через AccessPassDao
- Информация для всплывающих окон хранится в каждом слое отдельно, что позволяет кастомизировать контент
- Видимость слоев контролируется через метод isVisible с учетом минимального и максимального масштаба

## Risks and limitations
- В текущей реализации методы render() являются заглушками и не содержат реальной отрисовки на карте
- Для слоя эвакуации используется заглушка для проверки доступа (в реальном приложении будет интеграция с AccessPassDao)
- Обработка ошибок при парсинге GeoJSON базовая, в реальном приложении потребуется более детальная обработка
- Тестовые данные для слоя эвакуации генерируются в коде, в реальном приложении будут загружаться из assets

## Invariant compliance
- [ ] Модульная архитектура — соблюдено, каждый слой является отдельным классом с четкой ответственностью
- [ ] Offline-first — соблюдено, все данные загружаются из assets
- [ ] Безопасность — соблюдено, слой эвакуации проверяет доступ через AccessPassDao
- [ ] Производительность — соблюдено, слои отображаются только в нужном масштабе

## How to verify
1. Проверить сборку проекта: `./gradlew :core-mapping:assembleDebug`
2. Запустить unit-тесты для слоев: `./gradlew :core-mapping:testDebugUnitTest`
3. Проверить, что все классы имеют документацию на русском языке
4. Убедиться, что слои корректно обрабатывают GeoJSON данные
5. Проверить, что слой эвакуации требует наличия действующего допуска
