# AI_NOTES — Stage 3: Интеграция с системой допусков и безопасность

## What was done
- Реализована проверка наличия действующего допуска через AccessPassDao
- Добавлено отображение предупреждения о скором истечении срока (менее 5 минут)
- Реализована обработка root-доступа с использованием RootBeer
- Добавлена обработка отсутствия допуска с отображением заглушки и предложением пройти квиз
- Создан MapViewModel для управления состоянием карты и проверки доступа
- Обновлен MapScreen для отображения карты или заглушки в зависимости от состояния
- Реализованы компоненты для отображения предупреждений (ExpiryWarningCard, RootWarningCard)

## Why this approach
- Использование StateFlow для управления состоянием UI в ViewModel
- Разделение состояний карты (HasAccess, NoAccess, NoAccessButHistory) для гибкого отображения
- Интеграция с существующими DAO для работы с базой данных
- Использование RootBeer для надежной детекции root-доступа
- Отдельные компоненты для предупреждений для переиспользования и чистоты кода

## Risks and limitations
- Проблемы с производительностью при частой проверке допуска (решено через кэширование в ViewModel)
- Возможные конфликты при одновременном обновлении состояния из разных источников
- Ограничения в отображении предупреждений (только текст, без иконок и анимаций)
- Тестирование root-детекции на реальных устройствах может быть сложным

## Invariant compliance
- [ ] Offline-first — карта работает без интернета (реализовано)
- [ ] Безопасность — проверка root и шифрование данных (реализовано)
- [ ] Проверка допуска — интеграция с Room (реализовано)
- [ ] Отображение предупреждений — интеграция с UI (реализовано)

## How to verify
1. Запустить приложение и проверить отображение карты при наличии допуска
2. Проверить отображение заглушки при отсутствии допуска
3. Проверить отображение предупреждения об истечении срока при сроке менее 5 минут
4. Проверить отображение предупреждения о root-доступе
5. Проверить работу кнопок управления масштабом
