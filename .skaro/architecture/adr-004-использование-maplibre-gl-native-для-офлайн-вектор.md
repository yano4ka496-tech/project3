# ADR-004: Использование MapLibre GL Native для офлайн-векторного картографирования

**Status:** accepted
**Date:** 2026-04-23

## Context
Приложение SafePlant требует отображения карт промышленных объектов с возможностью масштабирования от 10×10 км до 10 метров. Карты должны отображать дороги, здания, опасные зоны и предустановленные маршруты. Все данные должны быть доступны офлайн.

## Decision
Принято использовать MapLibre GL Native (форк Mapbox GL) для отображения карт. Карты будут в формате MBTiles, конвертированных из PDF через QGIS/ogr2ogr. Это обеспечивает векторное отображение с поддержкой масштабирования и возможность отображения различных слоёв (дороги, здания, опасные зоны).

## Alternatives
1. **Использование растровых тайлов:** rejected because не обеспечивает достаточного качества при масштабировании до 10 метров, требует значительно больше места для хранения и не позволяет эффективно отображать векторные данные (дороги, здания).
2. **Использование Mapbox GL с ключом и онлайн-тайлами:** rejected because требует интернет-соединения для загрузки тайлов, что нарушает требование работы офлайн, и может быть дорогостоящим при большом количестве пользователей.

## Consequences
- Positive: Высокое качество отображения при любом масштабе, эффективное использование места для хранения (векторные данные занимают меньше места, чем растровые), возможность отображения различных слоёв информации, работа полностью офлайн.
- Negative: Требуется ручная конвертация PDF в MBTiles на этапе сборки приложения, сложность настройки стилей отображения для разных типов объектов, ограниченная функциональность по сравнению с онлайн-картами.
- Risks: Конвертация PDF в векторные данные может терять точность или детали; сложность обновления карт без выпуска новой версии приложения; возможные проблемы с производительностью на устройствах с ограниченными ресурсами.
