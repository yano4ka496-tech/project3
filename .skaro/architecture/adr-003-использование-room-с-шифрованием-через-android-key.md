# ADR-003: Использование Room с шифрованием через Android Keystore

**Status:** accepted
**Date:** 2026-04-23

## Context
Приложение SafePlant должно безопасно хранить чувствительные данные: вопросы квиза, цифровые допуски, соответствие QR-кодов координатам. Требуется обеспечить защиту этих данных от несанкционированного доступа, особенно на устройствах с возможностью root-доступа.

## Decision
Принято использовать Room (SQLite) с шифрованием через Android Keystore. Все данные в базе будут зашифрованы с использованием алгоритма AES/GCM. Для хранения чувствительных метаданных (дата истечения допуска) будет использоваться EncryptedSharedPreferences.

## Alternatives
1. **Использование обычной SQLite без шифрования:** rejected because не обеспечивает защиту данных при физическом доступе к устройству или root-доступе, что нарушает требования безопасности для приложения, работающего с опасными зонами.
2. **Использование EncryptedFile для всех данных:** rejected because создаёт сложность в управлении файлами и не обеспечивает таких же возможностей для запросов, как реляционная база данных.

## Consequences
- Positive: Высокий уровень защиты данных даже при root-доступе к устройству, возможность выполнения сложных запросов к зашифрованным данным, интеграция с существующими инструментами Android для работы с базами данных.
- Negative: Увеличение сложности реализации и отладки, потенциальное снижение производительности запросов из-за шифрования/дешифрования, зависимость от Android Keystore, который может быть недоступен на некоторых устройствах.
- Risks: Возможные проблемы с совместимостью между разными версиями Android при использовании Keystore; сложность миграции данных при смене алгоритма шифрования; риск потери данных при сбоях шифрования/дешифрования.
