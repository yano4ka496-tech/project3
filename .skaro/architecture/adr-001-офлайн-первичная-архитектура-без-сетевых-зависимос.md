# ADR-001: Офлайн-первичная архитектура без сетевых зависимостей

**Status:** accepted
**Date:** 2026-04-23

## Context
Приложение SafePlant должно работать в условиях отсутствия интернет-соединения, включая режим "в самолёте". Это требование определяет всю архитектуру приложения, так как все данные должны быть доступны локально без необходимости сетевых запросов.

## Decision
Принято полностью офлайн-первичное архитектурное решение. Все данные упаковываются в APK или загружаются из assets при первом запуске. Приложение не выполняет никаких сетевых вызовов (REST, gRPC, брокеры сообщений, push-уведомления).

## Alternatives
1. **Гибридный подход с кэшированием:** rejected because не гарантирует работу в режиме "в самолёте" и требует сложной логики синхронизации при восстановлении соединения.
2. **Постоянная проверка соединения:** rejected because увеличивает потребление батареи, ненадёжна в условиях нестабильного покрытия и не соответствует требованию работы без интернета.

## Consequences
- Positive: Полная автономность приложения, работа в любых условиях, включая отсутствие сигнала, низкое потребление батареи, отсутствие задержек ожидания сетевых ответов.
- Negative: Увеличение размера APK (до 500 МБ), необходимость полной переустановки приложения для обновления данных, невозможность динамического обновления контента без выпуска новой версии.
- Risks: Пользователи могут долго не обновлять приложение, что приведёт к устареванию данных; невозможность удалённого отзыва допусков или экстренных уведомлений.
